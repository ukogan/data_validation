Implementation Details:
  • Border Logic: Calculate absolute difference between sensor and BMS percentages for occupied and unoccupied states
  • Green Border: Divergence < 10% (good synchronization)
  • Yellow Border: Divergence 10-20% (minor timing drift)
  • Red Border: Divergence > 20% (poor synchronization, needs investigation)
  • Both Bars: Apply same border color to corresponding sensor and BMS bars
  • Tooltips: Enhanced hover text shows divergence percentage and status
Advantages:
  • No additional space required - uses existing bar elements
  • Immediate visual feedback on correlation quality
  • Works with both occupied and unoccupied states
  • Clear color coding follows standard traffic light pattern
  • Maintains all existing functionality and layout
Implementation Code: 
// Calculate divergence level for border styling
function getDivergenceBorderClass(sensorPercent, bmsPercent) {
    const divergence = Math.abs(sensorPercent - bmsPercent);
    if (divergence < 10) return 'divergence-good';
    if (divergence < 20) return 'divergence-warning';
    return 'divergence-poor';
}

// Apply to both occupied and unoccupied sides
const occupiedDivergence = getDivergenceBorderClass(
    sensor.sensor_occupied_percent,
    sensor.zone_occupied_percent
);
const unoccupiedDivergence = getDivergenceBorderClass(
    sensor.sensor_unoccupied_percent,
    sensor.zone_standby_percent
);
